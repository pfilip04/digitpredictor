About the Digit predictor

Your drawing is converted into a sequence of strokes—coordinates tracking every point your mouse dragged over the canvas. To match the input format expected by the machine learning model, the drawing is rescaled and redrawn on a 28×28 pixel canvas. This canvas is then saved as a .png image.

To send it to the model, the image is first converted to binary data, then encoded into a Base64 string made up of ASCII characters. This allows the image to be safely embedded in an API request.

On the server, the image is decoded from Base64 back into its original binary form, then converted back to an image format. The model then processes this image using its trained parameters to make a prediction.

Model Overview

This digit recognizer is powered by a lightweight convolutional neural network (CNN) trained on a combination of:

  • 85% MNIST digits
  • 15% custom handwritten digits (slightly pixelated to introduce noise for training)

Training was done over 100 steps (~40 minutes) with data augmentation techniques including:

  • Random rotations and translations
  • Dropout and batch normalization

Network Architecture

  • 2 convolutional layers
  • 2 pooling layers
  • 2 fully connected layers

While compact, this architecture performs well in many cases.

Known Model Limitations

  • A moderate tilt and stretching can severely impact accuracy
  • Can confuse 1 with 7
  • Occasionally struggles with 6, 5, and 9
  • A very small size may result in a less accurate prediction

Planned Upgrades

  • A larger training set with additional custom samples
  • Support for multi-digit identification
  • Recognition and evaluation of mathematical equations
